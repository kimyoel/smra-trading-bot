## 5. 왜 진입 타이밍을 "크로스 감지"로 바꾸는가?

### 5-1. 현재 코드의 문제

```python
# 현재 calc_adx()
def calc_adx(df):
    if adx_val > 25 and dip_val > dim_val:
        return "long"
```

이 코드는 **ADX>25인 모든 봉에서 "long"을 반환**한다.  
ADX가 25 이상으로 유지되는 기간이 수십 봉에 걸칠 수 있으므로,  
봇은 매 봉마다 동일한 진입 신호를 받는다.

심볼별 포지션 중복 차단(`executed_symbols`)이 있어 실제 2중 진입은 안 되지만:
- 불필요한 CPU 사용 및 로그 오염
- **포지션 청산 후 같은 조건으로 즉시 재진입** (의도하지 않은 동작)
- 백테스트와 실전의 **진입 시점 불일치** → 백테스트 결과 재현 불가

### 5-2. 백테스트의 진입 방식

백테스트 엔진(`indicators.py v3.8`)은 다음과 같이 동작한다:
1. 매 봉마다 `compute_signals(df)` 호출 → N×35 신호 배열 생성
2. **현재 봉에서 신호가 발생하면**, 다음 봉 시가에 진입으로 기록
3. 동일한 신호가 연속으로 발생해도 **이미 포지션이 있으면 무시**

실전 봇에서 이를 정확히 재현하려면:
- **신호의 "전환"(False→True)을 감지**해야 한다
- 이전 봉은 False였고 현재 봉이 True → 전환 = 신호 발생
- 이전 봉도 True이고 현재 봉도 True → 유지 중 = 신호 없음

### 5-3. 구현 방식: signal_generator 레벨에서 처리

**indicators.py는 수정하지 않는다.** 각 지표 함수는 현재 그대로  
"long"/"short"/False를 반환하게 둔다.

**signal_generator.py에서 이전 봉 vs 현재 봉을 비교:**

```python
for ind_name in indicators:
    fn = INDICATOR_MAP[ind_name]
    result_curr = fn(df)           # 마지막 봉(확정) 포함
    result_prev = fn(df.iloc[:-1]) # 마지막 봉 제외 (직전 봉까지)
    
    if result_curr and result_curr != result_prev:
        # 전환 감지 → 이 지표는 "새로운 신호"
        cross_results.append(result_curr)
    else:
        # 유지 중이거나 False → 이 지표는 신호 없음
        cross_results.append(False)
```

**장점:**
- indicators.py 함수 시그니처/로직 변경 없음 (안전)
- 백테스트와 동일한 진입 조건 재현
- 한 번 진입한 조건으로 포지션 청산 후 즉시 재진입하는 문제 해결

### 9-3. TP/SL 전달 흐름 (코드 근거)


## 10. 왜 Sharpe 기반으로 심볼 충돌을 해소하는가? (v2.5 추가)

### 10-1. 문제: 같은 심볼에 여러 전략 동시 신호

17개 전략 중 같은 심볼을 대상으로 하는 전략이 여러 개 있다:
- **BTC/USDT**: 7개 전략 (Sharpe 36.04 ~ 27.06)
- **XRP/USDT**: 6개 전략 (Sharpe 34.39 ~ 16.40)
- **ETH/USDT**: 4개 전략 (Sharpe 29.60 ~ 16.93)

시장 조건에 따라 여러 전략이 동시에 신호를 발생시킬 수 있다.

### 10-2. 기존 처리 방식의 문제

기존에는 `main.py`의 `executed_symbols` set으로 막았다:
```python
if symbol in executed_symbols:
    continue  # 이미 이 심볼로 주문 시도함
```

이 방식의 문제:
1. **arbiter 단계에서 이미 계산 완료** — 마진, TP/SL, 수수료 분석까지 한 뒤 main에서 버림 → 낭비
2. **어떤 전략이 선택되었는지 로그가 불명확** — 단순히 "스킵"만 표시

### 10-3. 해결: arbiter에서 Sharpe 기반 중복 제거

**signal_arbiter.py v2.5**에서 심볼별 Sharpe 최고 1개만 선택:

```python
# 1. 모든 후보를 Sharpe 내림차순 정렬
executable.sort(key=lambda x: x["sharpe"], reverse=True)

# 2. 심볼별 첫 등장만 유지 (= Sharpe 최고)
seen_symbols = {}
for sig in executable:
    symbol = sig["strategy"]["symbol"]
    if symbol in seen_symbols:
        # SKIP-SHARPE-CONFLICT 로그
        continue
    seen_symbols[symbol] = sig["strategy"]["id"]
    deduplicated.append(sig)
```

### 10-4. Sharpe를 기준으로 삼는 이유

Sharpe ratio = (수익률 - 무위험수익률) / 표준편차

**위험 조정 수익률**이므로:
- 높은 Sharpe = 같은 변동성(위험)에서 더 높은 수익
- 동일 심볼에 여러 전략이 경합할 때, Sharpe가 높은 전략이 **더 효율적**
- OOS 점수나 수익률만으로 비교하면 MDD(리스크)를 간과할 수 있음

예시: XRP/USDT 경합
```
XRP_1h_SHORT_AD       Sharpe 34.39  ← 승자
XRP_1h_SHORT_AD_2     Sharpe 30.00  ← 제외 (같은 지표, 다른 SL)
XRP_15m_SHORT_CMF_CCI Sharpe 17.85  ← 제외 (다른 TF/지표)
```
